Section 1Decision tree — which user are you?

A retail user arriving at Livermore is in exactly one of five mental states. The state is determined by what they bring, not who they are. Sophistication doesn't matter; starting context does.

User arrives
Landing page, deep link, returning visit, or chat — they're here.
What do they bring?
The single question that determines product flow.
Use case A
A portfolio
"I already own AAPL, MSFT, GLD…"
Use case B
A thesis
"I think energy will outperform"
Use case C
An idea
"What about a momentum strategy?"
Use case D
A saved strategy
"How's my old SPY trend doing?"
Use case E
Nothing yet
"Just browsing — what's hot?"

Why this segmentation works: each starting state implies a different entry door, a different primary flow, and a different module stack. Mixing them under one "build a strategy" funnel forces the portfolio holder, the thesis builder, and the casual browser through the same wizard — which works for none of them perfectly.

Section 2The 5 use cases — flows & modules

Each card: who the user is, what they say, the 5-step product flow, and the key modules each flow needs. Module status uses the same three pills as the rest of the framework docs.

A — "I have a portfolio" most retail investors
Priority 1 · High
"I own AAPL, MSFT, NVDA, and some SPY. What strategies make sense for what I already hold? Should I rebalance? Should I hedge?"

Product flow

  1. Import portfolio (CSV / broker connect / manual entry) Entry door 1: portfolio uploader
  2. See portfolio dashboard — current holdings, weights, factor exposure, behavior fingerprint "Your book is growth-tilted, 80% trending"
  3. Get strategy suggestions matched to this portfolio "Add a trend filter overlay on TLT to hedge"
  4. Backtest the suggestion as a portfolio overlay (not a replacement) "With this overlay, your 2022 DD would have been −12% instead of −24%"
  5. Subscribe to signal alerts for each holding Position-level signal monitoring

Key modules required

  • Portfolio import (CSV / manual)Missing
  • Broker API connect (paper)Missing
  • Portfolio analyzer (style, factor exposure)Missing
  • Asset behavior fingerprintMissing
  • Strategy-to-portfolio matcherMissing
  • Multi-holding signal monitoringShipped
  • Backtest engine (per-holding)Shipped
  • Signal alerts (email)Shipped

Gap reality: the engine + signal infrastructure to support this user exists, but the entry door doesn't. A "Connect your portfolio" path is the missing front end. Estimated to build: portfolio import + analyzer ≈ 2–3 weeks (CSV upload is the cheap path; broker connect is the hard one).

B — "I have a thesis" sophisticated retail / prosumer
Priority 2 · Medium-High
"I think small-caps are mispriced relative to large-caps after this rate cycle. I want to build a strategy that bets on the rotation."

Product flow

  1. Free-form thesis input (chat or text box) Entry door 2: thesis composer
  2. LLM parses thesis → asset class + direction + horizon "You're betting small-caps over large-caps over months"
  3. Suggest universes + strategy families that express the thesis "Try IWM/SPY pair, or factor-tilt to size + value"
  4. Show 2–3 template matches with thesis-aligned defaults "Sector rotation with size tilt; long/short via pair"
  5. Backtest as thesis validation (regime + counterfactual) "In 2008–2010, this worked; in 2020 it didn't"

Key modules required

  • Thesis input UI (text box / chat)Partial
  • Thesis → strategy parser (LLM-based)Partial
  • Asset class + direction extractorMissing
  • Thesis-to-universe translatorMissing
  • Template matcher with defaultsShipped
  • Counterfactual analyzer (regime)Missing
  • Backtest engineShipped

Gap reality: the chat builder (strategy_builder_iterate) does ~30% of this today. Real "thesis mode" needs a dedicated UI where the thesis is treated as a first-class object — not a chat message. Estimated build: 1–2 weeks (most heavy lifting is LLM prompt engineering + connector to existing templates).

C — "I have an idea" casual retail with a hunch
Priority 1 · High
"I want to try momentum on tech stocks." or "Show me a low-vol strategy on consumer staples."

Product flow

  1. Wizard (5 questions) or chat input Entry door 3: existing wizard
  2. Top 2–3 templates recommended with reasoning Already shipped — recommend()
  3. Pick template, enter universe, set risk preset 4-block summary UI
  4. Backtest with credibility warnings DataQualityService + _credibility_warnings
  5. Save / publish / monitor signals Full lifecycle present

Key modules required

  • 5-question wizardShipped
  • Template gallery (25 templates)Shipped
  • 4-block summary editorShipped
  • Risk preset → engine paramsShipped
  • Backtest engine + result viewerShipped
  • Signal composer (true custom)Missing
  • Asset behavior fingerprint at picker timeMissing

Gap reality: this is Livermore's existing flagship flow. ~85% complete. The two missing pieces (signal composer + behavior fingerprint) come from the previously-documented signal-composition workstream. Estimated to fully close: ~14 hours of engineering as per the build queue in the audit doc.

D — "I have a saved strategy" returning user
Priority 1 · High (retention)
"I built a sector rotation strategy two weeks ago. How's it doing? Did the signal change today?"

Product flow

  1. Sign in → personal dashboard (saved strategies list) Entry door 4: account dashboard
  2. Each strategy shows: current signal, last-week change, trailing return Per-strategy "live performance" tile
  3. Click in → full result viewer with comparison vs. last save History tab in workspace
  4. Iterate / fork / unsubscribe / re-publish Full lifecycle tools
  5. Receive email when any subscribed signal flips signal_alert.py cron-dispatched

Key modules required

  • Personal dashboard (saved strategies)Partial
  • Per-strategy live performance tileShipped
  • Signal change detection (cron)Shipped
  • Email alertsShipped
  • Comparison vs prior saveShipped
  • Strategy versioning (v1/v2/v3 lineage)Partial
  • Portfolio-style "my strategies" viewMissing

Gap reality: the backend infrastructure for this is excellent. The frontend has individual pieces but no unified "my strategies" hub. Estimated to ship a proper dashboard: 1 week (mostly Next.js page composition; data already accessible via existing endpoints).

E — "I'm just exploring" top-of-funnel browser
Priority 3 · Medium
"I heard about quant strategies. Show me what's working. Show me what other people are using."

Product flow

  1. Land on homepage / community feed Entry door 5: discovery surface
  2. Browse popular strategies + trending tickers "What's hot this week"
  3. One-click "try it" → backtest with defaults No friction; show, don't ask
  4. See result, optionally sign up to save Anonymous backtest endpoint exists
  5. Convert to use case C, D, or A based on engagement Funnel handoff

Key modules required

  • Community feedShipped
  • "Trending strategies" / "What's hot" surfaceMissing
  • Trending tickers surfacePartial
  • Anonymous backtest endpointShipped
  • One-click "try with defaults"Partial
  • Sign-up funnel after first backtestPartial

Gap reality: the foundations are there (community feed, anonymous backtest, market pulse). Missing is the "what's hot / what worked this week" curation layer — which is computable from existing data. Estimated: 3–5 days for a trending-strategies tile + landing-page reorg.

Section 3Priority matrix — where to invest

Qualitative call. No scoring, no weights — just where the leverage is highest given the current state of the product.

Priority 1 — Invest now
A — "I have a portfolio"
Largest retail segment. Entry door is missing. Backend is ready. Biggest TAM expansion you can do.
C — "I have an idea"
Existing flagship flow. 14 hours of work closes the last gap (signal composer + behavior fingerprint).
D — "I have a saved strategy"
Retention. Backend done; one week of frontend work to a real "my strategies" hub. Cheapest retention win available.
Priority 2 — Invest next quarter
B — "I have a thesis"
Smaller segment but high-value, high-engagement users. Mostly LLM prompt engineering plus a dedicated UI. 1–2 weeks.
Priority 3 — Defer or do small
E — "I'm exploring"
Top-of-funnel matters but the existing community feed + anonymous backtest cover the basics. A trending-strategies tile is a 3–5 day add-on, not a workstream.

Why this ordering

Three principles drive the ranking:

  1. Existing infrastructure wins. A use case where the backend is already shipped and only the entry door is missing (A, D) is much cheaper than one needing new engine work.
  2. Retention beats acquisition. A returning user with a saved strategy is worth 5–10 new visitors. Use case D is the cheapest of them all (no new backend code).
  3. "Custom build" only matters for users who got past the front door. The custom signal composer (which sits inside use case C) is a closer of the existing funnel — high leverage conditional on A and D pulling users in. Don't deepen C if A and D are leaking users at the entry.

Section 4PM recommendation: what to build next

Three concrete bets in priority order. Sized for a 6-week roadmap.

Bet #1

Portfolio entry door + analyzer

A CSV/manual portfolio uploader → dashboard that shows holdings, style, factor exposure, and recommends overlay strategies. Lights up the largest retail segment.
Ship by: ~3 weeks. Headcount: 1 fullstack + 0.5 designer. Risk: CSV is cheap; broker connect can be deferred to v2 (Plaid / Snaptrade later).
Bet #2

"My strategies" dashboard + signal alerts unification

Personal hub showing all saved strategies with current signal, trailing performance, change history. Cheapest retention win — backend already shipped.
Ship by: ~1 week. Headcount: 0.5 frontend. Risk: Lowest in the queue. Just composition of existing API endpoints.
Bet #3

Signal composer + behavior fingerprint

Closes the "Custom Build" promise (Use case C). Lets users actually customize WHEN IN / WHEN OUT. Plus a tiny service that classifies an asset as trending or mean-reverting before the user runs a backtest.
Ship by: ~14 hours. Headcount: 1 fullstack. Risk: Lowest engineering risk; all schema and engine changes are additive.

What I'd defer

  • Thesis mode (use case B). High-value but lower-volume. The chat builder covers it well enough today; a dedicated thesis UI is a Q2/Q3 investment.
  • Trending-strategies tile (use case E). Nice-to-have. Build only after the funnel from A and D is full — otherwise you're spending time on a marketing surface that no one converts from.
  • Walk-forward optimization, paper trading, options strategies, long/short engine. All institutional-tier. Live in the previously-documented Sprint 3 (multi-week each); don't pull these forward until A, D, and C-closer are shipped.

The single thing to watch for

Don't let "build features for each use case" cause use-case fragmentation. The five use cases must all funnel into the same backtest pipeline and the same saved-strategy lifecycle. The differences are at the entry door (steps 1–2 of each flow); the rest should converge. If any use case spawns its own backtest engine or its own save format, that's a flag — push back.
